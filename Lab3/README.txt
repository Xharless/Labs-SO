Detalles de uso del programa:
	- Para poder ejecutar el programa, debe tener en la misma carpeta archivo java, make y carpeta archivos_prueba.
	- Para correr el código, ejecutar el comando "make" en una terminal al mismo nivel de los archivos principales.
	- Una vez ejecutado mostrará por consola los resultados.
	

Consideraciones:
	- Nombre del archivo principal "Lab3.java". Nombre del archivo make "makefile". 
	- Para el correcto funcionamiento del programa, la carpeta "archivos_prueba" debe encontrarse al mismo nivel que el archivo Lab3.java
	- En caso de que se quiera seleccionar otra carpeta del tipo "Caso", este se debe cambiar manualmente en la variable "path" dentro del main.
	- Se asume que los archivos están en formato .txt
	- Se recomienda que si se quiere cambiar de carpeta para probar otro caso, se ejecute el comando "make clean" antes de probar otro caso.
	- Cuando se ejecuta el programa, en la salida por terminal se mostrará la ruta del archivo que se ejecutó, tanto por Hebras como por Forks.
	- Mostrará el tiempo que se demora tanto los Forks como las Hebras en encontar la letra (esta misma también se mostrará).
	- Una vez se ejecute todos los archivos, se entregará la clave completa y el tiempo total que se demoró en descifrarla por completo tanto para Forks y para Hebras.
	- En el archivo excel se encuentran hojas de cada Caso, cada una con una tabla y gráfico con los tiempos. También existe una hoja con el análisis cualitativo.
